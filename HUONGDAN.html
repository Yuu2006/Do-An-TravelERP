Hiểu dự án trong vài phút.

Tài liệu trực quan cho backend Spring Boot: mỗi vai trò làm gì, request đi qua những lớp nào, database liên kết ra sao và từng phân hệ sử dụng những API nào.

Backend Hoạt Động Như Thế Nào

Mỗi request cần đăng nhập đi qua các lớp dưới đây. Khi lỗi, hãy kiểm tra theo thứ tự từ trái sang phải.

1Máy khách

Frontend, Postman hoặc Swagger gửi request HTTP đến các đường dẫn /api/**.

2Bảo mật JWT

JwtAuthFilter đọc bearer token, Spring Security kiểm tra quyền theo prefix API.

3Controller

Controller gom endpoint theo nghiệp vụ: khách hàng, kinh doanh, sản phẩm, kế toán.

4Service

Service xử lý rule nghiệp vụ: đặt tour, thanh toán, voucher, phân công, quyết toán.

5Oracle DB

Repository lưu JPA entity được map rõ ràng tới table và column trong Oracle.

Luồng Nghiệp Vụ Chính

Đây là luồng dễ hiểu nhất để nắm cách hệ thống ERP du lịch vận hành từ đầu đến cuối.

1Xem tourKhách xem tour công khai tại /api/public/tour.
2Đặt tourKhách tạo đơn trong bảng DONDATTOUR.
3Thanh toánKhách thanh toán qua nhóm /api/thanh-toan.

Thiết Lập Database

Ứng dụng dùng Oracle và chỉ validate schema khi khởi động. App không tự tạo bảng và không tự seed dữ liệu.

Bước 1

Tạo schema

Chuẩn bị user/schema Oracle và khai báo thông tin kết nối trong file .env.

Bước 2

Chạy DDL

Chạy src/main/resources/db/KhoiTaoBang.sql để tạo lại bảng, constraint và index.

Bước 3

Seed dữ liệu

Chạy data_v1.sql, sau đó data_v2.sql. Mật khẩu đăng nhập seed là password.
